Git/I Love Turtles Directions

Check git is installed

  1. Open terminal
  2. run the git --version command, hit Enter.
  3. If you don’t see a version number you’ll need to install homebrew and then git (or call Jim or Tim over to help)

Sign Up For Github (version control) / Download Sublime Text (editor)

  1. Go to https://github.com/ and sign up / sign in with your optimising account
  2. Download and install Sublime Text (either using homebrew, or from the site itself)

Basic Unix Commands - See Folders

  1. Using terminal enter the command ls + Enter. This will list all of your files and folders in your root directory.

Basic Unix Commands - Make a Folder

  1. If you don’t already have a folder called “specialists”l, enter the command mkdir specialists + Enter. This will “make a directory” / create a folder called “specialists”

  1. To verify that you made it correctly enter ls + Enter. This will list the files in your root folder. You should see a folder named “specialists”

  1. Next we need to cd specialists - which means “change directory to the specialists folder”. The folder % should change names. ~ meant the root folder.

  1. Now enter pwd - this will “print working directory” - letting you know the path you’re in. On my computer at home it’s /Users/james/specialists

  1. If you want to open the folder in Finder, enter open . and it will open the folder/directory in the visual format. The full stop means “all files”.

Download the repository from Github

A repository is a group of files that make up a coding project. We need to download the “SEO Exercises” code onto our local machine (your work computer).

  1. Visit https://github.com/jimmy2/tech-seo-exercises.git in your browser
  2. Hover over the code and copy the repository url

  1. Go to terminal and enter git clone https://github.com/jimmy2/tech-seo-exercises.git - this will clone/copy the files to your local machine

  1. Confirm it’s successfully downloaded by typing in ls (list)

  1. Then change directories (cd tech-seo-exercises) - hint type in cd then tec then tab, it should autocomplete.
  2. Now you should be in the directory. Hit ls again to see all of the files.

  1. Open the directory in Sublime Text - File > Open > Users > Username > specialists > tech-seo-exercises

  1. Open the directory in Sublime Text to see all of the Files (subl .)

  1. Click on the index.html file and on line 6 and 21 change the name in the title tag and h1 from Jim to your name.

  1. Get an image of yourself and put it in the /images/ directory. Then edit the file name on line 14

  1. Save the file.

Add the files to your own git repository

  1. In terminal (make sure you’re in the /Users/yourname/specialists/tech-seo-exercises folder), enter git init to intialise a new git repository.

  1. Usually we’d also run git status to see what files have been modified

  1. Then you can run git add . (remember the full stop is all files) to add the files to your repository.

  1. Followed by git commit -m “Initial commit” - don’t forget the quotes! - this adds a message (Initial commit) to the version control log. It’s best practice to add a descriptive message to each major change you do (e.g. git commit -m “Added a new picture to the homepage”)

Set Up A Repository From Github

  1. Now it’s time to “push” or send our changed files back up to our github account, so we can create websites using it. In the top right hand corner of your github account, click the plus icon and select “new repository”

        

  1. Name the repository tech-seo-exercises. Then click Create Repository.

  1. Now you’ll receive instructions on how to push the files to the repository you’ve just created at github.com - follow the instructions under “…or push an existing repository from the command line” to push your code changes up to the server.

  1. Add the 3 lines of instructions to terminal

If you get an error message:

 try “git remote remove origin” and then try again ;)

You may be asked for your computer password to upload it.

  1. Otherwise, you may need to enter your github email (your optimising email) as well as a temporary access code which can be found at (Account > Settings > Developer Settings > Personal access tokens > Generate new token) . Ping Jim or Tim for help with this one bit cause it might be tricky. Select 90 days and check repo.

That access token will be your password when you “git push” for the initial time only.

  1. If all goes well, you will see your code pushed to github.

  1. You can also refresh the page and see your code with the “Initial commit” message.

Personalise your space and give your team a name

GIve Netlify Access to All repositories / or alternatively you can just give it access to the one repo.

Pick the Repo you want to go live

Make sure its deploying from the main branch

Vercel Deployment

Navigate to https://vercel.com/  and sign up for an account with your GitHub

Select Authorise Vercel

You should see this page, select add github account

Give access to all Repos, or alternatively you can select just one.

Import the Repo you wish to go live
